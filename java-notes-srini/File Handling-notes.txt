IO Streams
===================

--> To perform operations on the file we need to use IO streams

--> Using IO streams we can establish connection between java program and file.

		  java program<-----------------------------------------------> write File
          	 java program<-----------------------------------------------> read File


IO streams are divided into 2 types

	1) Byte stream       : To read/write binary data(images, video, audio)

	2) Character Stream  : To read/write character data(txt files)

--> Byte stream providing 2 types of classes

			1) Input Stream Related classes  (ex: FileInputStream )
			2) Output Stream Related classes (ex: FileOutputStream )


--> Character Stream providing 2 types of classes 

			1) Reader classes  (Ex : FileReader)
			2) Writer classes  (Ex : FileWriter)

Serialization
=================

The process of converting java object into file or network supported form is called serialization.

The process of converting file or network supported from into java object is called de-serialization.


--> when you are building distributed applications(micro-services, Kafka), object must be converted into a format that can travel over the network. By using serialization  we can make this as possible.

--> when you want to make our java class object as serializable our class must implement Serializable interface.


ex: 

class Person implements Serializable{
	
	int id;
	String name;
}

public class SerializableDemo {

	public static void main(String[] args) throws IOException, ClassNotFoundException {
		// TODO Auto-generated method stub
		
		Person p=new Person();
		p.id=102;
		p.name="Srini-it";
		
		
		 System.out.println("==============Serialization process starts==========");
		 
		
		FileOutputStream fos=new FileOutputStream("C:\\Users\\srini\\Desktop\\IN-Srini Java boot camp-Sep-2025\\02-core-java-programs\\java.io\\hello-ser.ser");		
		ObjectOutputStream oos=new ObjectOutputStream(fos);		
		oos.writeObject(p);		
		oos.close();
		System.out.println("==============Serialization process completed==========");
		
		
		System.out.println();
		System.out.println();	
		
		
		System.out.println("==============de-serialization process starts==========");
		
		
		FileInputStream fis=new FileInputStream("C:\\Users\\srini\\Desktop\\IN-Srini Java boot camp-Sep-2025\\02-core-java-programs\\java.io\\hello-ser.ser");
		
		ObjectInputStream ois=new ObjectInputStream(fis);
		
		Object obj=ois.readObject();		
		Person p1=(Person) obj;
		
		System.out.println(p1.id+"     "+p1.name);
		
		ois.close();		
		
		
		System.out.println("==============de-serialization process completed==========");
		
	}

}

What is SerialVersionUID 
============================

--> For every .class file JVM will generate a random number that is called serialversionUID

--> When we perform serialization on the java object JVM will assign .class file serialversionUID to serialized file.

--> When we de-serialize, JVM will compare both serialize file UID and .class file UID, if both are not matching de-serialization will not happen.

--> To Overcome the problem we have to manually generate the our own serialversionUID then JVM will not assign serialversionUID.


transient keyword
========================

--> Transient is a keyword in java which is used to restrict the serialization 

--> If we have any sensitive data/ secrets then we should not serialize those fields.



ex :

class Person implements Serializable{
	
	
	int id;
	String name;
	
	transient String password;
} 

note : If we serialize the above class(Person) object, id and name fields will be serialized, password will not be serialized because it is a transient variable.

--> while de-serializing  the file, the transient keyword variables printing the default values instead of actual values.
